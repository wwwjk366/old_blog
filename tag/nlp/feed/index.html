The syuzhet Package

The get_nrc_sentiment function of syuzhet package uses National Research Council Canada (NRC)’s sentiment dictionary to calculate the presence of eight different emotions and their corresponding valence in a text file. The output of this function will be a data frame where each row represents a sentence from the original file. The columns include one for each emotion type was well as a positive or negative valence. Let’s try some sampe examples first.

get_nrc_sentiment("what a lovely little young girl")

##   anger anticipation disgust fear joy sadness surprise trust negative
## 1     0            2       0    0   2       1        2     1        0
##   positive
## 1        2

Pretty neat, eh? Now we can apply it on our book

LOR1nrc <- cbind(parag = seq_along(LOR1), get_nrc_sentiment(LOR1))
LOR1nrc$negative = LOR1nrc$negative * (-1) # change negative score to negative
LOR1byline <- LOR1nrc %>% select(-positive, -negative) %>% 
  melt(id = c("parag"))
head(LOR1byline)

Generate Charts for Sentiment

In the above line chart I only looking at the first 100 paragraphs of the book. It’s useful but it’s too granular to do the analysis. Most of the cases we only care about the binary sentiment Positive or Negative, the get_sentiment functon of syuzhet provides the simple positive or negative scores using different kinds of methodology.

LOR1sentiment =data.frame(cbind(parag = seq_along(LOR1), senti = get_sentiment(LOR1, method = "afinn")))
head(LOR1sentiment)

##   parag senti
## 1     1     0
## 2     2     0
## 3     3     6
## 4     4    14
## 5     5    -1
## 6     6     4

The support method for this function includes

  • afinn from Finn and Nielsen
  • bing from Bing Liu and collaborators
  • nrc from Saif Mohammad and Peter Turney
  • standford from Manning, Christopher D., et al

Dr. Matthew Jockers did a little test by himself here and he found out that in his experiment, afinn and bing method yields the best result. So in this post I will use afinn method, but feel free to test different methods by yourself.

Fourier Transformation of Time

The above three plots are pretty but it is hard to compare them because they all have different length and the positive and negative sentiment are seprated so it is hard to see the cumulative sentiment flow. It is also hard to compare book to book because each book will have different length. But luckliy we have a tool called Fourier Transformation to normalize the length of the x-axis. Recall the formula below:

\[\hat{f}(\xi) = \sum_{-\infty}^{\infty}f(x)e^{-2\pi i x \xi}dx\]

The author of syuzhet package integrated this feature into the function called get_transformed_values and it also has a low pass filter. We can think low pass filter as a “noise filter”. The lower the number it is, the smoothier curve we will get. In this post I used low_pass_size = 3 parameter value, same as Dr. Jockers used in his paper.

tt = "Fourier Transformed Sentiment in 'The Fellowship of the Ring'"
cptn = "thinkdatascience.com | Michael Yan"

LOR1ft <- as.numeric(get_transformed_values(LOR1sentiment$senti, 
                                            low_pass_size = 3,
                                            scale_vals = TRUE,
                                            scale_range = FALSE))
LOR1ft <- data.frame(cbind(linenumber = seq_along(LOR1ft), ft = LOR1ft))

ggplot(data = LOR1ft, aes(x = linenumber, y = ft)) +
  geom_bar(stat = "identity", alpha = 0.8, color = "green3", fill = "green3") +
  labs(title = tt, caption = cptn, x = "Narrative Time", y = "Fourier Transformed Value") +
  theme( text = element_text(family = ff[58], size = 12),
        legend.title = element_blank(),
         plot.title = element_text(colour="black", size = 12, face = "bold"),
         strip.text.x = element_text(size=15, face="bold", colour = "black"),
         strip.text.y = element_text(size=25, face="bold", colour = "black"),
         legend.position="right")

Now the sentiment flow can be seen much clearly, and we can easily compare the different books
